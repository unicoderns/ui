# Buttons
Use custom theme to define button styles based on your own variants. Buttons support multiple sizes, state and more.

[Github repo](https://github.com/unicoderns/ui/tree/master/packages/button)

```html
    <ui-button
      :anchor="false" :toggle="false" :outline="false" :active="false" :variant="'primary'" :size="'sm'"
      :aria:role="'some role'"
      @toggle="toggle"
      >Button</ui-button>
```

## Props

| Name | Type  | Description |
| ---- | ---- | ----------- |
| `anchor` | `boolean` | if true the button is rendered as an anchor tag `<a>` |
| `toggle` | `boolean` | if true the button shows as state of active or inactive and sends the `toggle` event |
| `outline` | `boolean` | outline class is added to the button |
| `variant` | `string` | variant class defined in the theme |
| `size` | `string <ButtonSizeVariant>` | ButtonSizeVariant enum for sm, md or lg size.  |
| `active` | `boolean` | shows button as active adding the active class |

# Events
Button will emit this events while changing state

| Event | Params | Description |
| ---- | ---- | ----------- |
| Toggle | active: `boolean` | Will only raise when `toggle = true` |

## Slots
| Name | Description |
| ---- | ----------- |
| `default` | actual button content |

# Accessibility
Change the next properties to change the default accessibility text

| Name | Property | Default |
| ---- | ---- | ----------- |
| role | aria:role | `button` |

Check the page [what is aria](/docs/docs-aria--page#what-is-aria-config) to see how to config accessibility globally for buttons.

# Theme
Use the next `json` to define a new theme for the button

```json
{
  cssClass: {
    main: string, // main button class
    active: string, // css class for the active state
    outline: string, // css class for outline buttons
    variants: { // define a custom list of variants
      'variant1': string,
      'variant2': string,
      ...
      'variantN': string,
    },
    outlineVariants: { // must match the same variants as before
      'variant1': string,
      'variant2': string,
      ...
      'variantN': string,
    },
    sizes: { // provide clases for 3 size variants: sm, md, lg
      [SizeVariants.Large]: string,
      [SizeVariants.Medium]: string,
      [SizeVariants.Small]: string,
    },
  },
}
```

Check the page [what is a theme](/docs/docs-theming--page#what-is-a-theme) to see how to use this json
